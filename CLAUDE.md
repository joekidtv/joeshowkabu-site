# CLAUDE.md — Joe Show Kabu 制作・運用規範

このリポジトリで作業するときは、必ずこのファイルのルールに従うこと。
Joe Show Kabuは**中高生向け**の株式・金融教育Instagramアカウント。運営者は高校生・非エンジニア。

## リポジトリの基本

- 公開: **GitHub Pages**(https://joekidtv.github.io/joeshowkabu-site/)。`main` にpushすると自動反映される
  (Netlifyは無料ビルド枠超過のため停止中。README.md参照)
- サイト本体の構成・JSONの編集ルールは [README.md](README.md) が正。矛盾したらREADMEを優先
- **絶対に壊してはいけないもの**: `.github/workflows/`(市場データの毎日自動更新)、
  `scripts/update_market*.py`、`market.json` / `market_us.json` の `name` キー、
  news.json/lectures.jsonの既存エントリ

## 2つのコンテンツフォーマット

### ① レクチャー教材(フィード投稿・シリーズ「第◯回」)
- スタイル: **フラット・ネオブルータリズム(White Pop / New Brutalism)**
- キャンバス: **1080×1350px(4:5)** が標準。指示があれば1080×1080も可
- 純白背景 `#FFFFFF` + 左ボーダー帯 `#FF3333` 16px / 右ボーダー帯 `#33CC66` 16px(上下には付けない)
- フッター: スチールブルー `#4682B4` の帯(免責・出典・アカウント名の定位置)
- すべての箱要素: `border: 4px solid #000000` + `box-shadow: 8px 8px 0 #000000`(ぼかし0のハードシャドウのみ)
- `border-radius: 0`(装飾用の正円バッジのみ例外)

### ② Joe Show NEWS(ストーリーズ・毎日更新)
- キャンバス: **1080×1920px(9:16縦型)**
- ヘッダー: **青グラデーション**(`#1E3A8A` → `#3B82F6`、上から下)。
  ※グラデーション禁止ルールの唯一の例外。NEWSのヘッダー以外では使わない
- カード類のオフセットシャドウ: **サーモンピンク `#FA8072`**(`box-shadow: 8px 8px 0 #FA8072`)
- それ以外(白背景・黒4px枠・Noto Sans JP)はレクチャーと共通

## 共通デザインルール(絶対厳守)

- 色パレット: 赤 `#FF3333` / 緑 `#33CC66` / 白 `#FFFFFF` / 黒 `#000000` 基調。
  補助色はNEWSの青グラデ・サーモンピンク、フッターのスチールブルーのみ
- **グラデーション禁止**(NEWSヘッダーのみ例外)、ぼかしシャドウ禁止、半透明の装飾禁止、パステルカラー禁止
- グレー(#333等)への逃げは不可。文字・線は純黒 `#000000`
- フォント: **Noto Sans JP**(Google Fontsから読み込み)。本文weight 700以上、見出し900推奨
- 見出し64〜96px・`letter-spacing:-0.02em`目安 / 本文28px以上(出典表記のみ20〜24px可)
- **1スライド1メッセージ。本文は60文字以内**。収まらなければスライドを分割する
- 絵文字は1スライド最大1個(図版内も合算)。既定は0個
- 赤=リスク・値下がり側 / 緑=メリット・値上がり側、という色の意味を崩さない

## 免責事項(必須)

- 各エピソードの最終スライド(CTA)に必ず以下の統一文言を入れる:
  > ※投資は自己責任でお願いします。本コンテンツは情報提供を目的としたものであり、特定の金融商品の勧誘・推奨ではありません。
- NEWSスライドではフッターに短縮版「※投資は自己責任で。特定銘柄の推奨ではありません」を常設
- 定位置: スライド下部、左右ボーダー帯の内側、20〜24px、黒

## トーン(中高生向け)

- 断定表現の禁止:「上がる」「買うべき」「絶対」「確実」→「〜する傾向がある」「〜という考え方がある」等に言い換える
- メリットとリスクは必ず両論併記。片側だけのスライド構成にしない
- すべてのファクト(株価・統計・制度)に出典表記 `出典: 発行元名 (年)` を入れる。出典不明の数値は使わない
  (無い場合は `出典: [要確認]` を置いてユーザーに報告)
- 金融系の文言を含むスライドは、公開前に `financial-content-compliance` スキルでチェックする

## マスコット

- **黄金クロソ(ゴールデンクロス)**: 知識・ポジティブ担当。メリット・複利・NISA活用の回に登場
- **デッドクロス**: リスク管理・注意喚起担当。値下がりリスク・損切り・詐欺注意の回に登場
- メリットとリスク両方を扱う回では**必ず両方を登場させる**(フェアネスのため)
- 同一スライドに両方置くときは、左(赤側)=デッドクロス、右(緑側)=黄金クロソ
- 画像素材が無いときは黒枠プレースホルダー(「黄金クロソ画像」等)を置く。勝手にSVGで描き起こさない

## スライド制作ワークフロー

詳細なデザイン仕様は `joe-show-kabu-design` スキルにあるので、スライド生成時は必ずそれを併用する。

1. **HTML生成**: 1スライド=1ファイルで `slides/lectures/episode-N/slide-01.html`(NEWSは
   `slides/news/YYYY-MM-DD/slide-01.html`)に保存。CSS込みの単一ファイル、
   `body` 直下にキャンバス原寸(1080×1350等)の `.slide` 要素1つ。scale縮小はしない
2. **PNG書き出し**(Chromeヘッドレス使用・追加インストール不要):
   ```bash
   python3 tools/export_slides.py slides/lectures/episode-N assets/lectures/episode-N --format lecture
   python3 tools/export_slides.py slides/news/YYYY-MM-DD assets/news/YYYY-MM-DD --format news
   ```
3. **PNGを目視確認**(Readツールで開き、文字化け・レイアウト崩れ・トークン逸脱がないか見る)
4. **JSON更新**: レクチャーなら `lectures.json` の**末尾**に1件追記(サムネイルは
   `assets/lectures/episode-N/slide-01.png` を指定可)。NEWSなら `news.json` の**先頭**に追記。
   形式はREADME.mdの記載に厳密に従う。既存エントリは変更しない
5. **コミット**: `git add` → `git commit` まで実行する。
   **`git push` は実行しない**(このMacにはpush用の認証情報が無いため)。
   commitが終わったら「GitHub Desktopでpushをお願いします」とユーザーに伝えて作業を終える
6. Instagramへの投稿は**手動運用**(ユーザーがPNGを使って投稿する)。自動投稿API連携はしない

### 書き出し環境メモ
- この環境にNode.jsは無い。Puppeteer/Playwrightは使わず、
  `/Applications/Google Chrome.app` のヘッドレスモード(`--headless=new --screenshot`)を使う
- ラッパーは `tools/export_slides.py`(Python3標準ライブラリのみ。pip install不要)
- Webフォント読み込みのため `--virtual-time-budget=10000` を指定済み。ネット接続が必要
