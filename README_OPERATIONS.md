# Joe Show Kabu 日常運用マニュアル(Claude Codeへの頼み方)

このファイルは、Claude Code(チャットでお願いすると作業してくれるAI)に
「どう言えば何をしてくれるか」をまとめた、あなた(運営者)向けのメモです。
むずかしい操作は全部Claudeがやるので、あなたは日本語でお願いするだけでOKです。

---

## よく使うお願いの例

### 1. レクチャースライドを作ってほしいとき

> 「第10回『分散投資とリスク許容度』のスライドを作って。本編は6枚くらいで」

こう言うと、Claudeが自動で:
1. いつものデザイン(白背景・赤緑の帯・黒枠)でスライドHTMLを作る
2. PNG画像(Instagram投稿用サイズ)に変換して `assets/lectures/episode-10/` に保存する
3. サイトのレクチャー一覧(lectures.json)に第10回を追加する
4. GitHubに送信(push)して、公式サイトに自動反映させる

あなたがやるのは、**できあがったPNGをInstagramに手動で投稿するだけ**です。
PNGファイルの場所はClaudeが最後に教えてくれます(Finderで開いてもらうことも可能)。

### 2. NEWSスライドを作ってほしいとき

> 「今日のNEWSスライドを作って。内容は『日経平均が3日続伸、半導体株が牽引』で」

こう言うと、ストーリーズ用の縦長(9:16)デザインでPNGを作り、
`assets/news/今日の日付/` に保存して、news.jsonにも追加してくれます。

### 3. 直しをお願いしたいとき

> 「第10回の3枚目、文字が多いから2枚に分けて」
> 「タイトルを『◯◯』に変えて、PNGも作り直して」

スライドのHTML(設計図)がリポジトリに残っているので、何度でも直せます。
直したら自動でPNGも作り直してくれます。

### 4. サイトだけ更新したいとき

> 「第9回のインスタ投稿URLが決まったので、lectures.jsonに入れて公開状態にして」
> 「news.jsonに今日の見出しを追加して」

### 5. 投稿文(キャプション)もほしいとき

> 「第10回のインスタ用キャプションとXの投稿文も作って」

---

## 覚えておくと安心なこと

- **「push(プッシュ)して」= サイトに公開して**、という意味です。
  pushすると数分で https://joekidtv.github.io/joeshowkabu-site/ に反映されます
- 毎日の市場データ(日経平均など)は**何もしなくても自動で更新されます**。
  Claudeにもこの仕組みには触らないよう指示済みです
- Instagramへの投稿は今まで通り**あなたが手動**で行います(自動投稿はしていません)
- スライドのデザインルール(色・文字数・免責事項など)は `CLAUDE.md` に書いてあり、
  Claudeは毎回それを守って作ります。ルール自体を変えたいときは
  「CLAUDE.mdのルールを変えたい。◯◯を△△にして」とお願いすればOK

## うまくいかないときの言い方

- 画像の文字がおかしい/レイアウトが崩れている
  → 「◯枚目のPNG、ここが崩れてるので直して」(スクリーンショットを貼るとより確実)
- サイトに反映されない
  → 「pushできてるか確認して」(数分待っても出ないときはこれ)
- 間違って公開しちゃった
  → 「さっきのpushを取り消して」(Claudeが安全に元に戻します)

## ファイルの場所(参考・触らなくてOK)

| 場所 | 中身 |
|---|---|
| `slides/` | スライドの設計図(HTML)。直しのときに使う |
| `assets/lectures/episode-N/` | 第N回の投稿用PNG |
| `assets/news/日付/` | NEWSの投稿用PNG |
| `lectures.json` / `news.json` | サイトに表示される一覧データ |
| `CLAUDE.md` | Claudeが守るデザイン・運用ルール |
| `tools/export_slides.py` | HTMLをPNGにする道具(Claudeが使う) |
